# FTHexMerge 使用说明书

> 版本：V1.0.0.1 ｜ 官网：www.fortiortech.com

## 一、软件简介

**FTHexMerge** 是一款嵌入式芯片 **HEX 文件合并工具**，用于把 BOOT、APP、Pflash（DFlash 数据）等多个固件文件，按地址合并成**单个 `Full.hex`**，供烧录器一次性烧录。

典型用途：固件发布、产线烧录前合并、BOOT+APP 一体化烧写。

**核心能力**

- 解析标准 Intel HEX（I8/I16/I32）与 DAT 二进制数据文件
- 字节级冲突检测：自动识别「相同 / 冲突 / 独占」
- 按优先级自动仲裁 + 冲突段手动改选
- 输出标准 I32HEX（含扩展地址、起始地址、EOF），并**逐字节回读校验**
- 自动忽略「到 DFlash 的跨区间隙」，避免输出膨胀

---

## 二、运行环境

- 操作系统：**Windows 10 / 11**（需 .NET Framework 4.8，系统通常自带）
- 运行方式：双击 `FTHexMerge.exe`
- 无需安装，绿色运行

---

## 三、快速开始（三步合并）

1. **启动** → 弹出「文件选择」窗口。
2. **添加文件**：点「浏览…」选择，或直接把文件**拖进窗口**（自动按顺序填入空槽）。
3. 点「**比较**」→ 进入比较窗口 → 菜单「**合并 → 保存 Full.hex…**」→ 预览确认 → 选择保存路径。

完成。状态栏会显示「已保存到 …（回读校验通过）」。

## 四、界面与操作

### 4.1 文件选择窗口（第一步）

| 区域 | 说明 |
|---|---|
| 第 1 / 2 / 3 组文件 | 三个文件槽。**第 3 组可选**（两文件比较时不填即可）。每个槽支持 HEX 或 DAT |
| 浏览… | 弹出文件选择框（支持 `*.hex`、`*.dat`） |
| 拖拽 | 把文件拖到窗口任意位置，自动按空槽顺序填入 |
| DAT 基地址 | 仅 `.dat` 文件显示。文件名含 `0x…` 时**自动预填**，可手动修改（十六进制，如 `0xE00000`） |
| ↔ 交换 | 组 1↔2、组 2↔3、组 1↔3 互换位置（影响优先级默认顺序） |
| 结果摘要 | 点「比较」后显示各文件字节数、段数、地址范围、冲突数 |
| 比较 / 取消 | 「比较」进入比较窗口；「取消」（或 Esc）关闭窗口 |

> **至少选择 2 个文件**才能比较。

### 4.2 比较窗口（第二步）

窗口从左到右：**定位栏 + 比较视图 + 滚动条**，顶部为菜单栏，底部为状态栏。

**菜单栏**

| 菜单 | 功能 |
|---|---|
| **文件** | 重新比较…（返回文件选择窗，保留上次文件）／ 退出 |
| **合并** | 优先级 ▸ ／ 空隙填充 ▸ ／ ── ／ 保存 Full.hex… |
| **导航** | 上一个冲突 ／ 下一个冲突（到头/尾自动回绕） |
| **帮助** | 关于 FTHexMerge… |

**定位栏（左侧鸟瞰）**

- 整列压缩显示所有行，颜色以**第一个文件**为准（冲突黄、缺失灰、选定珊瑚）
- 中间半透明灰色「拇指」表示当前可见区，**可直接拖动**定位栏快速跳转

**比较视图（中部自绘网格）**

- 每个文件一列块：上方显示**文件名 + 各段信息**（`Block N Start … Ends … Length …`），下方是「地址 + 16 字节」
- 行背景与字节着色含义见 [六、配色说明](#六配色说明)

**交互**

| 操作 | 效果 |
|---|---|
| 鼠标滚轮 / 拖滚动条 / 拖定位栏 | 滚动浏览 |
| **↑ / ↓ 方向键** | 上下移动选中行（自动滚入视口） |
| 左键点行 | 选中该行（浅蓝高亮） |
| 右键点**冲突行** | 弹出段级菜单：「保留 文件X」或「恢复按优先级」 |

### 4.3 合并预览（保存前）

菜单「**合并 → 保存 Full.hex…**」后弹出预览窗：

- **段个数 / 合并后大小 / 覆盖地址范围 / 间隙** 摘要
- **输出段列表**：段与间隙各自成行，间隙行末标 `(间隙)`
- 若有**到 DFlash 的跨区间隙**：浅蓝提示块列出地址区间，说明「已忽略，默认不填充」
- 若有**超过 1MB 阈值的间隙**：浅黄提示块列出，说明「未填充（避免输出膨胀）」
- 「确定合并」→ 选保存路径；「取消」→ 返回

### 4.4 关于

菜单「**帮助 → 关于**」：显示程序名与版本、公司官网。

## 五、功能详解

### 5.1 优先级（菜单：合并 → 优先级）

多文件在同一地址都有数据且**值不同**（冲突）时，按优先级自动决定保留谁。默认顺序即文件槽顺序（组 1 优先）。可菜单切换为「文件2优先」「文件3优先」等。

### 5.2 空隙填充（菜单：合并 → 空隙填充）

合并结果中相邻段之间的**地址空洞**（间隙）如何处理：

| 选项 | 行为 |
|---|---|
| 不填充 | 间隙保持空洞（输出最小） |
| 0xFF | 间隙填 `0xFF` |
| 0x00 | 间隙填 `0x00` |

**填充阈值 = 1MB**：单段间隙超过 1MB 视为「跨大区」，**一律不填**（否则 HEX 文本会膨胀到几十 MB，甚至撑爆内存）。详见 [八、注意事项](#八注意事项)。

### 5.3 冲突处理

- **自动**：按优先级（5.1）决定冲突字节保留哪个文件。
- **手动**：右键冲突行 →「保留 文件X」可对该冲突段强制保留指定文件；「恢复按优先级」撤销手动选择。
- 手动选择以**珊瑚红**标记，与优先级自动选择视觉区分。

### 5.4 到 DFlash 的间隙（自动忽略）

BOOT/APP 通常在 PFlash（低地址），DAT 数据在 DFlash（高地址，由基地址决定）。两者之间的大间隙是**跨 flash 区**的，烧录时各自烧、无需在 HEX 里填满。本工具：

- 自动识别 DAT 基地址为 DFlash 起点
- 落点跨进 DFlash 的间隙**默认忽略、不填充**
- 在合并预览与状态栏**明确提示**「已忽略到 DFlash 的区间 [地址]」

### 5.5 键盘操作

| 按键 | 功能 |
|---|---|
| ↑ / ↓ | 上 / 下移动选中行 |
| Esc | 文件选择窗：关闭窗口 |

## 六、配色说明

着色风格参考 WinMerge，便于快速识别行状态：

| 颜色 | 含义 |
|---|---|
| 🟨 黄绿 `#EFCB05` | 该行存在冲突（多文件值不同） |
| ⬜ 灰 `#C0C0C0` | 该文件在此地址无数据（缺失） |
| 🟥 珊瑚 `#EF7774` | 该段已被手动选定保留此文件 |
| 🟦 浅蓝 `#ADD6FF` | 当前选中行 |
| 🟫 浅黄褐 `#F1E2AD` | 冲突字节且未选中（按优先级将保留其它文件） |

## 七、输出文件格式（Full.hex）

合并输出为**标准 Intel HEX（I32HEX）**：

- 按地址从小到大排序
- 每条数据记录 **16 字节**（末条可不足）
- 地址 > 64KB 时自动插入 **类型 04 扩展线性地址**记录
- 含 **类型 05 起始执行地址**记录（程序入口，按优先级取首个含 05 的文件）
- 文件末尾追加 **EOF**（`:00000001FF`）
- **逐字节回读校验**：写出后再解析一遍，与合并结果逐字节比对，零差异才判定保存成功

## 八、注意事项

1. **DAT 文件必须填基地址**：DAT 是纯二进制，第 i 字节对应地址 `基地址 + i`。基地址错会导致 DFlash 数据落到错误地址、甚至与 BOOT 冲突。文件名含 `0x…` 时会自动预填，请核对。

2. **大间隙不会填满**：空隙填充有 1MB 阈值。BOOT 与 DFlash 之间的几十 MB 间隙（跨区）不会被填——这是有意为之，避免输出膨胀到几十 MB 并撑爆内存。如需了解某间隙为何未填，看合并预览的提示块。

3. **再次打开合并出的 Full.hex**：可把它作为输入文件之一再次比较（例如核对合并结果）。

4. **重新编译前先关闭程序**：开发场景下，运行中的 `FTHexMerge.exe` 会占用文件，导致重新生成失败。

5. **手动改选会保留**：右键「保留 文件X」的手动选择会记入合并结果；「恢复按优先级」可清除。
