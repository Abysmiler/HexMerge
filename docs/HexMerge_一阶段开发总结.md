# HexMerge 一阶段开发总结

> 周报素材。完整版见下，末尾附「极简版」供直接复制。

## 项目一句话

**HexMerge** —— 嵌入式芯片 HEX 文件合并工具，将 BOOT / APP / PFlash(DAT) 多路固件合并为单一 Full.hex。
技术栈 WPF (.NET Framework 4.8) + MVVM（CommunityToolkit.Mvvm），100% 自研、仅用宽松许可依赖。

---

## 一、核心功能（已交付）

1. **多格式解析与合并**
   - Intel HEX 全格式（I8/I16/I32，记录 00–05）、DAT 二进制（DFlash）
   - 流式解析，大文件不占爆内存
2. **字节级冲突仲裁**
   - 优先级自动仲裁 + 段级手动改选
   - **「无」优先级模式**：强制逐个确认冲突段，未确认则阻止合并（防漏处理）
3. **自绘可视化比较**
   - 自绘 `FrameworkElement`（零 DataGrid），像素级平滑滚动，彻底消除卡顿
   - 差异三态着色 + 冲突导航浮窗（计数 / 优先级 / 上、下一个冲突跳转）
4. **合并预览与输出**
   - 段列表 + 摘要（段数 / 大小 / 范围 / 间隙）、DFlash 跨区识别、空隙填充（0xFF/0x00，阈值保护）

## 二、近期工作重点（适合本周报）

- **UI 主题统一**：抽 `Theme.xaml` 共享资源字典（配色 + 样式），文件选择窗 / 关于 / 浮窗 / 合并预览 / 所有弹窗统一为卡片式风格（主蓝 + 圆角 + 投影 + 主题按钮）

- **细节打磨**：按钮加粗、主蓝加深、状态栏文字、空隙填充默认 0xFF、关于对话框卡片化
- **打包发布**：Release V1.0.0.1，分层目录结构（exe + config + bin/依赖库 + UserConfigFile 工作区 + README）

## 三、技术亮点 / 难点攻克

| 难点 | 解决方案 |
|------|----------|
| 大间隙填充 OOM（14MB → 40MB+ / 560MB 内存） | 1MB 阈值 + 超限不填并提示 |
| 渲染异常触发循环弹窗（恶性） | 空值兜底 + App 级熔断（去重 + 5 次/5 秒退出） |
| 万行比较滚动卡顿 | 自绘只画可见行，零绑定零触发器 |
| 冲突跳转边界（段内行误跳本段首） | `HasConflict` 跳过当前段 |
